Important Note about Authentication

⚠️ Authentication is currently in development mode:

- Each request without an auth token gets a DIFFERENT user_id
- This means conversations created in one request won't appear in GET requests
- To see your conversations, you need to use the same auth token for both POST and GET
- Or use the WebSocket connection which maintains the session
